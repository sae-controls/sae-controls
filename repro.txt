Reproducibility note
====================

This file pins everything required to reproduce the published numbers.
The frozen artifacts in `artifacts/` are what the paper cites; the
hashes below are what you should expect after running the pipeline
end-to-end on a single GPU. The CPU-side analysis stages (the
decomposition scripts and figure rendering) are deterministic from the
artifacts.


Software environment
--------------------

The published runs used these exact pinned versions. Nothing in the
pipeline depends on these exact versions; nearby versions should work
if your environment differs, with the caveat that frozen-artifact
hashes may shift slightly under different BLAS / CUDA stacks.

  Python      3.11
  torch       2.5.1
  transformers 4.46.3
  huggingface_hub 0.26.5
  accelerate  1.1.1
  datasets    3.0.2
  numpy       1.26.4
  scipy       1.14.1
  tqdm        4.67.1
  matplotlib  3.9.2   (paper figures only)

Hardware (published runs):
  NVIDIA A40 (48 GB), CUDA 12.4, bf16
  Single-GPU is sufficient for every script.

Datasets
--------

AmbigQA (Min et al., 2020)
  HF repo:    sewon/ambig_qa
  Splits:     train (10,036) + validation (2,002)
  Filtered to multipleQAs with ≥ 2 short-answer (≤ 3 word) disambiguations.

TriviaQA-Web (Joshi et al., 2017)
  HF repo:    mandarjoshi/trivia_qa
  Config:     rc.web.nocontext (no evidence documents — we use only
              question + alias list; the full rc.web pulls ~10 GB of
              evidence files we never read)
  Splits:     train + validation

WikiText-2 raw (Merity et al., 2017)
  HF repo:    Salesforce/wikitext
  Config:     wikitext-2-raw-v1, split=test
  Used as the out-of-distribution reference for shuffle controls and for
  the position-mode polysemy partition.

Determinism notes
-----------------

The slot-detection and forward-pass stages use bf16 on GPU and may
exhibit small numerical drift across CUDA versions. The downstream
discrete decisions (which alias the model commits to, which features
the score function selects) are stable in practice but not guaranteed
bit-identical. The downstream analysis stages (decomposition, paired
tests, figures) are deterministic from the encoded SAE activations and
will reproduce hashes exactly given the same input artifacts.

Random seeds:
  SEED            = 0    (data sampling, random-feature controls,
                          per-pair shuffle draws within AmbigQA)
  SHUFFLE_SEED    = 42   (WikiText-shuffled draws; intentionally
                          distinct so that AmbigQA-shuffle and
                          WikiText-shuffle streams do not correlate)
  RANDOM_BASELINE_SEED = 1  (5-random-feature interpretability baseline,
                          src/config.py; disclosed for completeness)
